Delay Prediction Methodology

Forecasts delays 20 minutes into the future using real-time GTFS data and historical patterns.

How It Works

1. Delay Trend (last 30 min)
Analyzes if delays are worsening, stable, or improving

2. Route Patterns (historical data)
Checks if this route typically has delays at this time of day

3. Prediction Formula
Future Delay = Current + Trend + Route Pattern

Confidence Scores (50-95%)

  • High (85-95%): Extensive data, consistent patterns
  • Medium (60-80%): Moderate data availability
  • Low (50-55%): Limited data or unpredictable behavior

Urgency Levels

  • URGENT: 15+ min, rapidly worsening
  • Severe: 12+ min or large increase
  • Moderate: Monitor and prepare resources
  • Improving: Delay decreasing
  • Stable: Minor delays, standard monitoring

Understanding Problem Routes

Shows routes experiencing significant delays right now based on the last 1,000 delay records (~60-90 seconds of data).

Metrics

Current Delay (e.g., "16 min delay (current)")
Average delay for this route in the last ~90 seconds. Only shows routes with delays > 3 minutes.

Recent Data Points (e.g., "142 recent data points")
Number of delay records for this route in the last 1,000 system-wide records. Higher counts = more active vehicles.

Last Updated
Shows when data was last refreshed (auto-updates every 30 seconds).

Key Points

  • Real-time snapshot: Last ~90 seconds, NOT historical averages
  • For historical patterns: See "Delay Hotspots" panel
  • Sorted by severity: Worst delays shown first
  • Click any route: Jumps to map view

Understanding Delay Hotspots

Identifies recurring delay patterns by analyzing the last 50,000 delay records (hours to days of historical data).

How It Works

1. Groups by Route + Hour
Combines delays by route AND time of day (e.g., "Route 15 at 8:00")

2. Calculates Averages
Finds typical delay for each route+hour pattern

3. Filters & Ranks
Shows only patterns with avg delay > 3 min, sorted by severity

Metrics

Route + Hour (e.g., "Route 15 at 17:00")
This route typically has delays at this time (24-hour format)

Average Delay (e.g., "18 min avg")
Typical delay severity for this route at this hour

Frequency (e.g., "(67x)")
Number of occurrences. Higher = more reliable pattern
‚Ä¢ 50+ = Very reliable
‚Ä¢ 20-50 = Moderate confidence
‚Ä¢ <20 = Possible pattern, limited data

Use Cases

Example: "Route 15 at 17:00: 18 min avg (67x)"

  • Route 15 consistently delayed at 5 PM
  • Schedule extra buses proactively
  • Pre-position backup resources
  • Warn passengers about typical delays

vs. Problem Routes

Delay Hotspots: Historical patterns (hours-days) for proactive planning
Problem Routes: Real-time snapshot (~90 sec) for immediate response
